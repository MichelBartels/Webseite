PHP-Kontrollstrukturen

Wie jede Programmiersprache besteht auch PHP aus vielen verschiedenen Kontrollstrukturen, die man bestenfalls alle kennen sollte. Deswegen werden auf dieser Webseite als Einstieg die wichtigsten vorgestellt. Dabei ist wichtig anzumerken, dass uns leider das Vorstellen aller Kontrollstrukturen nicht möglich ist, weswegen wir hier nur für die weiteren Kontrollstrukturen auf eine andere Webseite verweisen, die sich genauer mit dem Thema beschäftigt.

If/If-else

Wenn man auch nur einigermaßen komplexe Webseite erstellen will, wrid man feststellen, dass es in den seltensten Fällen sinnvoll ist, dass der gesamte Code ungeachtet bestimmter Bedingungen komplett von oben nach unten ausgeführt wird. Um mit solch Bedingungen umzugehen, wurde in PHP und auch in fast allen anderen Programmiersprachen die Möglichkeit einprogrammiert, bestimmte Code-Zeilen mithilfe der if(-else) Kontrollstruktur nur unter bestimmten Bedingungen auszuführen. Die Syntax dieser Struktur ist sehr einfach und ähnelt anderen Programmiersprachen sehr stark. Sie fängt mit „if“ an, worauf in Klammern die Bedingung folgt, die in einfacher Boolescher Algebra in der PHP-spezifischen Schreibweise geschrieben ist. Danach schreibt man nur noch in geschweiften Klammern, den für den Fall, dass die Bedingung wahr ist, auzuführenden Code. Wahlweise kann man noch ein „else“ mit dem Code in geschweiften Klammern, der ausgeführt werden soll, falls die Bedingung nicht zutrifft. Dazu folgendes Beispiel:

In diesem Beispiel geht es um ein Gewinnspiel, das immer am 1. Tag eines Monats stattfindet. Als erstes wird geguckt, ob heute der Tag, ist an dem es stattfindet. Wenn es nicht stattfindet, gibt es zwei Möglichkeiten: Wenn es der 30. oder der 31. des Monats ist, wird einem gesagt, dass es nicht mehr lange bis zum nächsten Gewinnspiel dauert. Sonst wird einem mitgeteilt, dass es noch ein Bisschen bis zum nächsten Gewinnspiel dauert. Wenn man genau guckt fällt auch auf, dass eine alternative Syntax erwähnt wird. Damit keine Verwirrungen entstehen werden hier nochmal die verschiedenen Arten der Syntax übersichtlich gezeigt:

if (Bedingung){
    Befehle(, die ausgeführt werden, wenn die Bedingung stimmen, mit Semikolon jeweils am Ende)
} else {
    if(andere Bedingungen){
        Befehle(, die ausgeführt werden, wenn die andere Bedingung stimmt und die erste nicht, mit Semikolon am Ende)
    } else {
        Befehle(, die ausgeführt werden, wenn beide Bedingungen nicht zustimmen, mit Semikolon am Ende)
    }
}

Zusätzlich gibt es noch mehrere Möglichkeiten, die Syntax zu vereinfachen. Ganz vereinfacht sähe es so aus:


if (Bedingung):
    Befehle(, die ausgeführt werden, wenn die Bedingung zustimmen, mit Semikolon jeweils am Ende)
elseif (andere Bedingungen):
    Befehle(, die ausgeführt werden, wenn die andere Bedingung stimmt und die erste nicht, mit Semikolon am Ende)
else:
    Befehle(, die ausgeführt werden, wenn beide Bedingungen nicht zustimmen, mit Semikolon am Ende)
endif;

Der While-Loop

Irgendwann, wenn man von dem bloßen Arbeiten mit Bedingungen genug hat und sich an komplexere Projekte wagt, hat man das Probleme, dass man bestimmten Code leicht verändert oder gleich wiederholen will, um dies zu realisieren, könnte man theoretisch in den meisten Fällen einfach den Code kopieren. Dies ist allerdings suboptimal, da man dann Probleme hat, wenn man beispielsweise in jeder Wiederholung etwas ändern will oder die Datei irgendwo hochladen will und dann Probleme mit der Dateigröße hat oder wenn man die Anzahl an Wiederholungen von irgendetwas anderem abhängig machen will. Zu Behebung dieses Problems gibt es in PHP und vielen anderen Programmiersprachen die Kontrollstruktur des Loops(Wiederholung). Mit zwei davon wird sich auf dieser Seite beschäftigt. Die einfachste Form des Loops ist der while-Loop, der einen in geschweiften Klammern angegebenen Code ausführt solange eine davor in Klammern nach while angegebene Bedingungm erfü ist. Wenn man den Code einmal ausführen will, selbst wenn die Bedingung nicht stimmt kann man den Do-while-Loop verwenden, der sich von dem While-Loop nur dadurch unterscheidet, dass man anstelle des while am Anfang mit einer Bedingung nur do schreibt und while und die Bedingung nach den Code setzt. In diesem Beispiel geht es darum tausendmal eine Zahl auszugeben, die bei jeder Wiederholung um eins größer wird. Hierbei ist es nicht möglich den Code zu vereinfachen, da dies von dem Do-while-Loop nicht unterstützt wird. Allerdings kann man den While-Loop auch vereinfachen und deswegen wird im Folgenden nocheinmal die Syntax gezeigt:
Zuerst von dem Do-While-Loop:

do {
    Code(, der ausgeführt wird, solange die gleich geschriebene Bedingung stimmt, mit Semikolon am Ende)
} while (Bedingung);
Nun der nicht vereinfachten While-Loop:
while (Bedingung) {
    Code(, der ausgeführt wird, solange die Bedingung stimmt, mit Semikolon am Ende)
}
Nun vereinfacht:
while (Bedingung):
    Code(, der ausgeführt wird, solange die Bedingung stimmt, mit Semikolon am Ende)
endwhile;
Hierbei ist allerdings zu sagen, dass der vereinfachte Code nicht unbedingt einfacher ist.

Der For-Loop

Die Art, auf die der Loop eben geschrieben wurde, ist etwas umständlich. Genau deswegen gibt es den sogenannten For-Loop, der fast keinen Unterschied gegenüber dem While-Loop bis auf den Vorteil, dass man vor die Bedingung, durch ein Semikolon davon abgetrennt, Code schreiben kann, der ausgeführt wird, bevor der Code, der in der geschweiften Klammer steht, ausgeführt wurde, und nach die Bedingung, davon auch mit einem Semikolon abgetrennt, Code, der nach jeder Wiederholung ausgeführt wird, schreiben kann, hat. Auch bei dem For-Loop gibt es wieder eine vereinfachte Syntax, die jetzt hier zusammen mit der normalen nocheinmal neu aufgeführt wird:
Zuerst die normale Version:

for (Code, der als allererstes ausgeführt wird; Bedingung; Code, der nach jeder Wiederholung durchgeführt wird) {
    Code(, aus dem jede Wiederholung vor allem besteht, mit Semikolon am Ende)
}
Nun die vereinfachte Syntax:
for (Code, der als allererstes ausgeführt wird; Bedingung; Code, der nach jeder Wiederholung durchgeführt wird):
    Code(, aus dem jede Wiederholung vor allem besteht, mit Semikolon am Ende)
endfor;

Auch hier kann man sagen, dass die vereinfachte Version nicht sehr viel einfacher als die normale ist.

Die hier angewandten Einrückungen sind nicht für das Funktionieren des Codes notwendig, sie erleichtern aber das Lesen und Verstehen des Codes immens. Wir haben den Code so eingerückt, wie es weitläufig üblich ist.
